## ue-perfectworld 0.1.1 - 2026-07-17

- 教学代码的注释覆盖保持全面，但每条注释默认压缩为一行 `//`。
- 只有生命周期、所有权、网络权威、副作用或失败行为无法一行准确说明时，才使用连续多行 `//`。
- 最终自审新增注释简洁度检查，清理块状长文、语法复述、重复名称和冗余总结。

## ue-perfectworld 0.1.0 - 2026-07-17

- 新增项目中立的 Unreal Engine 规划、排查、教学实现、直接实现、审查和 Blueprint 协作工作流。
- 默认保持 UE 工作区只读，只有用户明确授权后才直接修改文件。
- 在对话中按用户实际编写顺序提供完整、可逐字录入且带教学注释的代码。
- 新增网络权威、事务安全、UI 参数契约、编码安全和项目惯例可信度规则。
- 新增阻断式最终自审，要求修复已知 BUG 并重写明显过度设计的实现。
- 新增独立品牌图标、正式发布元数据和 MIT License。

## 0.4.0 - 2026-07-16

### 执行效率

- 将 23 个 skill 的发现描述缩短，降低每轮固定上下文消耗。
- 将角色正文中的长方法论拆分为 22 份按需 playbook，完整能力保留，需要时只读取相关章节。
- 新增共享执行策略，要求先搜索再读取、复用项目事实、压缩成功日志并增量更新计划。
- 每轮只选择一个主角色；其他专业视角默认使用简短检查项。
- 多 Agent 只用于独立且并行收益高于协调成本的工作。

### 验证策略

- 默认使用 `ITERATION`，只验证本轮改动范围和受影响依赖边界。
- 引入 `LOW`、`MEDIUM`、`HIGH` 风险分级，风险只扩大局部验证范围，不自动触发全仓测试。
- 复用仍然有效的验证结果，并记录使结果失效的代码、配置、依赖或环境变化。
- 只有明确最终交付、发布、全量测试或最终验收时才进入 `FINAL_DELIVERY`。

### 发布流程

- 安装 Git for Windows，并使用 GitHub SSH 密钥进行长期认证。
- 新增 `scripts/publish-git.ps1`，用于生成更新记录、提交并推送仓库。

## 0.3.0 - 2026-07-16

- 将长项目测试拆分为 `ITERATION` 和 `FINAL_DELIVERY` 两种模式。
- 中间开发轮次禁止因为小修改、子任务完成或角色切换而自动运行全量测试。
- 23 个角色和 47 份继承参考资料全部加入验证范围优先级规则。
