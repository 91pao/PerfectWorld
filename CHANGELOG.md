## ue-perfectworld 0.1.4 - 2026-07-23

- 将证据、所有权、可维护性和失败恢复要求合并进项目一致性、完整实现和最终自审的权威规则，删除 3 份重复 reference。
- 新增 `ue-complete-implementation.md`，统一管理只读对话式完整实现、跨系统设计确认、代码完整性、Blueprint/编辑器步骤和验证要求。
- 将工作流规则改为“核心加按需”模型：网络、事务、UI 参数、编码、注释日志和 bug 修复规则只在相关任务中加载。
- 将路由器收敛为纯工作流选择，不再预加载会由已选技能再次读取的 reference。
- 统一使用“完整实现”，同时保留按真实依赖顺序提供完整代码、编辑器操作和验证步骤的行为。

## ue-perfectworld 0.1.3 - 2026-07-22

- 为非简单 UE 功能、跨系统集成、架构工作和完整教学实现新增证据门禁。
- 要求实现前建立所有权图，明确入口、运行时所有者、权威数据源、生命周期、网络边界、持久化、C++/Blueprint 分工和清理路径。
- 为跨系统教学实现新增协作检查点：先展示证据、所有权图、与参考实现的差异和最小设计，再提供完整可逐字录入代码。
- 新增单一事实源、真实调用者、空生命周期覆写、推测性抽象、需求变更测试、功能删除测试和无聊天上下文测试等可维护性检查。
- 新增失败恢复门禁：连续出现编译、链接、反射、资产或配置失败时，回溯架构和集成路径，不继续堆叠局部补丁。
- 将新门禁接入规划、局部实现、直接实现、问题排查、代码审查和 Blueprint 协作工作流。

## ue-perfectworld 0.1.2 - 2026-07-22

- 补充客户端上报动作数据的服务端验证规则，明确目标列表、命中位置和工具类型只能作为输入，不能作为动作真实发生的证明。
- 补充冷却起算时点和移动目标权威规则，避免把不同计时语义或客户端驱动位置误判为完整服务端验证。
- 补充异步事件身份快照规则，防止当前选择变化后篡改已发起事件的类型、伤害、范围、冷却或所有权上下文。
- 补充冗余状态与推测性 API 约束，要求验证真实 C++/Blueprint 调用者并保持选择与执行职责分离。
- 补充最终自审的结论范围检查，明确区分正常客户端复现修复、风险缓解和完整服务端权威实现。

## ue-perfectworld 0.1.1 - 2026-07-17

- 代码的注释覆盖保持全面，但每条注释默认压缩为一行 `//`。

- 最终自审新增注释简洁度检查，清理块状长文、语法复述、重复名称和冗余总结。

## 0.4.0 - 2026-07-16

### 执行效率

- 将 23 个 skill 的发现描述缩短，降低每轮固定上下文消耗。
- 将角色正文中的长方法论拆分为 22 份按需 playbook，完整能力保留，需要时只读取相关章节。
- 新增共享执行策略，要求先搜索再读取、复用项目事实、压缩成功日志并增量更新计划。
- 每轮只选择一个主角色；其他专业视角默认使用简短检查项。
- 多 Agent 只用于独立且并行收益高于协调成本的工作。

### 验证策略

- 默认使用 `ITERATION`，只验证本轮改动范围和受影响依赖边界。
- 引入 `LOW`、`MEDIUM`、`HIGH` 风险分级，风险只扩大局部验证范围，不自动触发全仓测试。
- 复用仍然有效的验证结果，并记录使结果失效的代码、配置、依赖或环境变化。
- 只有明确最终交付、发布、全量测试或最终验收时才进入 `FINAL_DELIVERY`。

### 发布流程

- 安装 Git for Windows，并使用 GitHub SSH 密钥进行长期认证。
- 新增 `scripts/publish-git.ps1`，用于生成更新记录、提交并推送仓库。
